# Claude Agent Guide for Knowledge Vault

이 파일은 `vault` 폴더(지식 데이터베이스) 내부에서 작업할 때 에이전트(Claude)가 반드시 준수해야 하는 규칙과 컨텍스트를 정의합니다.

## 🧠 System Context & Mission
- 이 `vault`는 사용자의 개인적/업무적 자산이 모이는 **인공지능 에이전트의 두뇌**입니다.
- **핵심 목표:** `1_Raw_Assets` 폴더의 가공되지 않은 원본 데이터를 읽고 분석하여, 사용자가 쉽게 탐색할 수 있는 정제된 마크다운 문서(`2_Wiki_Output`)를 생성 및 연결하는 것입니다.

## 📂 Directory Structure Rules
- `1_Raw_Assets/`: 수정 절대 금지 구역 (읽기 전용). 여행 사진(jpg), 엑셀(xlsx), 일기장(docx, md) 등 원본 자산 위치.
- `2_Wiki_Output/`: 가공 및 생성 구역. AI가 분석한 리포트, 사진 설명 맵, 위키 문서들이 저장되는 위치.
- `3_System/`: 인덱스 및 에이전트 웹사이트 연동용 데이터 저장 위치.

## 🛠️ Data Processing & Link Guidelines (카파시 스타일)
1. **원본 보존 (Source of Truth):** `1_Raw_Assets` 내부의 파일은 절대 임의로 수정하거나 삭제하지 마십시오.
2. **이모지 경로 처리:** 폴더 및 파일명에 이모지(📷, 📊, ✍️)와 공백이 포함되어 있으므로, 셸 명령어 및 경로 탐색 시 반드시 경로 전체를 큰따옴표(`""`)로 감싸야 합니다. (예: `cd "1_Raw_Assets/📷 여행사진"`)
3. **옵시디안 호환성:** `2_Wiki_Output`에 정제된 문서를 작성할 때는 문서 간의 연결을 위해 옵시디안 스타일의 위키링크(`[[문서이름]]`)를 적극적으로 생성하십시오.
4. **미디어/엑셀 추상화:** - 이미지(`jpg`) 파일을 분석할 때는 메타데이터, 촬영 날짜, 특징을 묘사한 `.md` 요약본을 위키 구역에 만드십시오.
   - 엑셀(`xlsx`) 파일을 요약할 때는 핵심 데이터를 마크다운 표(Table)나 `csv` 형태로 변환하여 위키에 기록하십시오.

## 💬 Conversation & Persona Command
- 사용자와 대화할 때는 친절하고 전문적인 개인 비서(Agent)로서 응대하십시오.
- 사용자가 과거 기억(여행, 일기)이나 업무 데이터를 물어보면, 항상 `1_Raw_Assets`에서 관련 파일이 있는지 먼저 스캔한 뒤에 이를 기반으로 답변을 재구성하십시오.
